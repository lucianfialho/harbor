Harbor uses `Effect.fn("name")` to add automatic OpenTelemetry spans to every critical operation. When you connect a trace backend, your pipelines become fully visible — no manual instrumentation needed.

## What you see in traces

```
harbor/ContactsDestination.write ──────── 8.4s
  ├─ harbor/HubSpotContacts.batchUpsert ─ 1.2s   batch 1/100
  ├─ harbor/HubSpotContacts.batchUpsert ─ 0.9s   batch 2/100
  │    └─ retry (429) ─────────────────── 0.3s
  └─ harbor/HubSpotContacts.batchUpsert ─ 1.1s   batch 3/100

harbor/LocalStage.write ─────────────── 38ms
harbor/LocalStage.read ──────────────── 12ms
harbor/GcsStage.write ───────────────── 2.4s
harbor/CsvSource.count ─────────────── 2ms
```

## Named spans across Harbor

| Span name | Where |
|-----------|-------|
| `harbor/ContactsDestination.write` | Top-level pipeline write |
| `harbor/HubSpotContacts.batchUpsert` | Each 100-record batch to HubSpot |
| `harbor/LocalStage.write` | Write JSONL to local filesystem |
| `harbor/LocalStage.read` | Stream JSONL from local filesystem |
| `harbor/LocalStage.exists` | Check if stage key exists |
| `harbor/GcsStage.write` | Write to Google Cloud Storage |
| `harbor/GcsStage.read` | Stream from Google Cloud Storage |
| `harbor/GcsStage.exists` | Check if GCS key exists |
| `harbor/CsvSource.count` | Count CSV lines |

## Connect a trace backend

Effect uses standard OpenTelemetry. Add your exporter in `main.ts`:

```typescript
import { NodeRuntime } from "@effect/platform-node"
import { Effect, Layer } from "effect"
import { OTLPTraceExporter } from "@opentelemetry/exporter-trace-otlp-http"
import { NodeTracerProvider } from "@opentelemetry/sdk-trace-node"
import { BatchSpanProcessor } from "@opentelemetry/sdk-trace-base"

// Setup OTel provider
const provider = new NodeTracerProvider()
provider.addSpanProcessor(
  new BatchSpanProcessor(new OTLPTraceExporter({ url: "http://localhost:4318/v1/traces" }))
)
provider.register()

// Your pipeline
const program = Effect.gen(function*() {
  const source = CsvSource({ path: "contacts.csv", schema: ContactSchema })
  const dest   = ContactsDestination({ token: process.env.HUBSPOT_TOKEN! })
  return yield* dest.write(source.stream.pipe(Stream.map(transform)))
})

NodeRuntime.runMain(program)
```

## Effect.fn — how it works

Every operation in Harbor uses `Effect.fn("span/name")`:

```typescript
// Before — no observability
const write = (key: string, lines: Stream<string, StageError>) =>
  Effect.gen(function*() { ... })

// After — automatic tracing span
const write = Effect.fn("harbor/LocalStage.write")(
  function*(key: string, lines: Stream<string, StageError>) { ... }
)
```

`Effect.fn` adds a span automatically. No middleware, no decorators — just the name.

## Adding spans to your own connectors

If you're [building a connector](/docs/guides/new-connector), use `Effect.fn` for every public method:

```typescript
export function MyApiSource(config: { token: string }) {
  // Named span for the count operation
  const count = Effect.tryPromise({ ... }).pipe(
    Effect.withSpan("harbor/MyApiSource.count")
  )

  return {
    stream: ...,
    schema: MyRecordSchema,
    count,
  }
}

// For service methods (generator functions):
const batchUpsert = Effect.fn("harbor/MyDestination.batchUpsert")(
  function*(batch: ReadonlyArray<MyRecord>) {
    // ...
  }
)
```

## Console tracing (dev)

During development, use Effect's built-in console tracer — no backend needed:

```typescript
import { Effect, Logger } from "effect"

const program = pipe(
  myPipeline,
  Effect.withSpan("main"),
  Effect.provide(Logger.minimumLogLevel(Logger.Level.Debug))
)

Effect.runPromise(program)
```
